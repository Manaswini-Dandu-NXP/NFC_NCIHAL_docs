getServicesAidCacheSize

Map<String, Integer> getServicesAidCacheSize (int UserID, 
                String category)

This api returns the CATEGORY_OTHER (non Payment)Services registered by the user along with the size of the registered aid group. This api has to be called when aid routing full intent is broadcast by the system.

This gives the list of both static and dynamic card emulation services registered by the user.

This api can be called to get the list of offhost and onhost cardemulation services registered by the user.

  • If the category is CATEGORY_PAYMENT than null value is returned.
  • If there are no non payment services null value is returned.

Parameters
UserID int: The user id of current user
category String: The category i.e. CATEGORY_PAYMENT , CATEGORY_OTHER
Returns
Map<String, Integer> The hashMap of Component Name of Non Payment Services and size of the registered aid group
Throws
IOException

updateServiceState

int updateServiceState (Map<String, Boolean> serviceState)

This api is called by applications to update the service state of card emualation services.

This api is implemented for CATEGORY_OTHER.

Requires NFC permission.

  • This api should be called only when the intent AID routing table full is sent by NfcService.
  • The service state change is persistent for particular UserId.
  • The service state is written to the Xml and read before every routing table change.
  • If there is any change in routing table the routing table is updated to NFCC after calling this api.

Parameters
serviceState Map: Map of ServiceName and state of service.
Returns
int whether the update of Card Emulation services is success or not. 0xFF - failure 0x00 - success
Throws
IOException if any exception occurs during the service state change.
